DPU Implementation tutorial

Project preparation

Copy module project into new directory.
Change groupId and artifactId in pom.xml.

Implementation class detail

Implement class module.Module. The implementation must extends just one of the following class:

  • cz.cuni.xrg.intlib.commons.module.GraphicalExtractor
  • cz.cuni.xrg.intlib.commons.module.GraphicalLoader
  • cz.cuni.xrg.intlib.commons.module.GraphicalTransformer

Function implementation details: (for more information please refer to the documentation)

  • getType - must return type according to implemented interface
  • getConfigurationComponent - return configuration component for given DPU
  • getSettings - return current DPU setting. If the configuration component is active return configuration from dialog.
  • setSettings - set setting for module and configuration component if exist

getSettings can return null in case of the presence of invalid configuration in active configuration component.
setSettings can throw ConfigurationException ,if the given configuration is invalid.

The configuration component must be located directly in package gui.

Configuration component

Configuration component must not contains any close button. The component will be closed from application.
Usage of any class from other package vaadin.ui is not possible.

Recommended practice

Use enum (like module.Config) to define names of object stored in configuration.
The configuration component (gui.ConfigDialog) should contains methods for set/getConfiguration for example setConfiguration and getConfiguration.

The configuration component can be developed in separate project and then add into project and connected by proper implementation of setConfiguration and getConfiguration method.

Start with sample implementation in module project and modify it to get your own DPU rather than starting just with pom.xml file. Just follow TODOs. :)
On the end do not forget to change the author.
